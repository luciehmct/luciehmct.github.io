    Detection of Back-Transitions from H Mode to L Mode in a Tokamak

    Swiss Plasma Center

    Context: In a tokamak, plasma can exist in a Low (L) confinement mode or a more stable, High (H) confinement mode. This project focuses on detecting the "back-transition" from H mode to L mode, a critical event to understand for maintaining plasma stability. The objective was to use machine learning to identify these transitions from complex time-series data and analyze the conditions that lead to them.

    What I implemented

    • Data Exploration & Selection: Analyzed and segmented raw experimental data, separating 35 unusable experiments (due to missing features like DML) to create a clean 'selected experiments' dataset for training.
    • Signal Preprocessing Pipeline: Developed the Python preprocessing pipeline to handle the complex time-series data. This included linear interpolation for missing values, applying Fast Fourier Transform (FFT) to the Halpha13 signal as an optional feature, and normalizing features.
    • Model Development: Implemented, trained, and evaluated time-series models to classify transitions, primarily a Bidirectional-LSTM (Bi-LSTM) and a Unidirectional-LSTM (Uni-LSTM), using PyTorch and TensorFlow.
    • Feature Engineering: Defined and tested multiple feature sets (e.g., features1, features2) combining parameters like plasma energy (Wtot), magnetic signals (DML), and density/temperature profiles.
    • Hyperparameter Optimization: Used Optuna to automate hyperparameter tuning to improve model performance.

    My contributions

    My role in this project was to help build the end-to-end machine learning pipeline. I was heavily involved in exploring the raw tokamak shot data, writing the Python scripts to clean and preprocess the signals, and engineering the feature sets for the models. I implemented both the Uni-LSTM and Bi-LSTM architectures, customized their time-window parameters, and ran a systematic evaluation to determine the best-performing model and feature combination.

    Results

    We systematically evaluated both models against different feature sets and preprocessing options. The Bi-LSTM model clearly outperformed the Uni-LSTM, achieving a top F1 Score of 0.7163 on the test set, as shown in the summary of best results below.
